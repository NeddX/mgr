43  /**
44  * @brief The packet header. Contains the type and length of the incoming payload.
45  *
46  * @details When a network transaction happens, the sending side always first sends @ref PacketHeader which
47  * contains the payload size, since @re PacketHeader has a fixed size, the receiving side first knows
48  * how many bytes to wait for before receiving the payload which has variable length.
49  * After receiving the packet header, the receiving side then decodes the packet to determine how many bytes
50  * is the payload.
51  * The bytes coming after a @ref PacketHeader is guaranteed to be the payload itself.
52  * */

54  {
55  PacketType type = PacketType::NoOp;
